Add ESOE LDAP Authentication and Identity Handler

Below you can add details about your deployed LDAP compliant directory service. This is the default backend that the ESOE makes use of to perform verification of user accounts and retrieve user identity.

An LDAP compliant directory is not required for the ESOE to operate, you can easily plug in databases, web services, flat files or anything else, we provide this as a simple means to get your initial site configuration running

Be aware if you choose not to configure an LDAP compliant directory service below You MUST perform, advanced manual configuration of your generated WAR before deployment $ldapForm
